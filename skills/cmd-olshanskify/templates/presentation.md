# Olshanskify Template: Presentations

Use when editing slide decks, talk outlines, conference abstracts, or lightning-talk notes.

> **Living template.** Seed rules only. Update as concrete talks and feedback reveal the user's actual on-stage preferences.

## Structure

- **Lead with the "so what".** First slide after the title states why the audience should care — in one sentence.
- **One idea per slide.** If a slide has two points, split it.
- **Arc shape:** Problem → tension → approach → evidence → takeaway. Don't bury the takeaway.
- **Closing slide is a call, not a "Thanks!".** Ask a question, share a link, or name the next step.

## Slide Content

- **Few words, big.** Slides are billboards, not documents. If you need a paragraph, it goes in the notes, not on the slide.
- **Code snippets** stay short (≤ ~10 lines). Highlight the single line that matters.
- **Diagrams over bullet lists** when explaining systems or flow. Build them up in stages across multiple slides rather than revealing all at once.
- **Numbers need comparators.** "2.3× faster than X" lands; "2.3× faster" does not.

## Voice (Speaker Notes + Abstracts)

- **Conversational, not academic.** Contractions are fine. Hedging ("perhaps", "arguably") usually isn't.
- **One clear thesis per talk.** The abstract, the opening, and the closing should all state it.
- **Self-deprecation sparingly** — it builds trust once; after that it undercuts authority.

## Formatting Consistency

- Same font sizes for the same slide roles across the deck.
- Same image treatment (drop shadow, border, or neither — pick one).
- Consistent code theme; never mix light and dark code blocks in a single deck.

## Before You Ship the Deck

Check:

1. Can a stranger skim the slides only (no audio) and still get the thesis? If not, the slides under-deliver.
2. Is there at least one memorable line or visual the audience will quote afterward?
3. Any slide that exists "for completeness" but has no speaker purpose? Delete it.
4. Does the closing slide give the audience something to do?
